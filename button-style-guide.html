Proposed Button Systems (Choose One)

Both options eliminate the conflict of having two different red buttons. Red is reserved exclusively for destructive/danger actions.

Option A: Teal Primary RECOMMENDED

btn-primary (Teal) - Main CTAs
btn-secondary (Gold) - Secondary actions
btn-gray - Neutral actions
btn-danger (Red) - Destructive only
Example: Active Module Card
Example: User Management
Example: Week Actions
Class Color Use For
btn-primary Teal Save, Submit, Create, Participate, Approve
btn-secondary Gold Cancel, Edit, Back, Manage, Reply
btn-gray Gray Save Draft, Archive, Dismiss
btn-danger Red Delete, Reject, Remove

Option B: Gold Primary + Light Secondary

btn-primary (Gold) - Main CTAs
btn-teal - Active/Launch actions
btn-secondary (Light Gray) - Secondary actions
btn-danger (Red) - Destructive only

Comparison Summary

Aspect Option A (Teal Primary) Option B (Gold Primary)
Primary CTA color Teal - calming, professional Gold - warm, attention-grabbing
Number of button types 4 (primary, secondary, gray, danger) 4 (primary, teal, secondary, danger)
Secondary button Gold - still prominent Light gray - very subtle
Brand color usage Teal prominent, Gold secondary Gold prominent, Teal for special actions
Visual hierarchy Clear: Teal > Gold > Gray > Red Gold and Teal compete for attention
Simplicity Simpler - fewer decisions needed More nuanced - need to choose between gold/teal
